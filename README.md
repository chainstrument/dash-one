 

# 📌 Projet : Personal Dashboard  

## 🎯 Objectif

Créer un **dashboard personnel web** qui regroupe des outils du quotidien :

* to-do list
* horloge
* météo
* citation du jour
* (plus tard : habits tracker, stats, personnalisation)

Le but principal est **d’apprendre en vibecoding avec un agent IA + GitHub Issues**.

---

# 🧠 Méthode de travail (très important)

## Structure GitHub

### 1. Repository

```
personal-dashboard
```

---

### 2. GitHub Project (Kanban)

Colonnes :

```
Backlog → Ready → In Progress → Review → Done
```

---

## 🧩 Organisation du projet

### EPICS (grands blocs logiques)

Les Epics servent uniquement à organiser :

* **Infrastructure & UI**
* **Productivité**
* **Informations personnelles**
* **Persistance**
* **Personnalisation**
* (optionnel plus tard : habits, analytics, backend)

👉 Les Epics ne sont PAS des tâches à coder directement.

---

## 📌 ISSUES (vraies tâches à coder)
 
Exemples :

* Créer le layout principal
* Créer le composant Widget
* Widget Horloge
* Widget Météo
* Widget Citation
* Ajouter une tâche (todo)
* Supprimer une tâche
* Compléter une tâche

---

## ✅ Acceptance Criteria

Chaque issue doit avoir :

* une description claire
* des critères de validation précis

Exemple :

```
Widget Horloge

- affiche heure réelle
- affiche la date
- se met à jour automatiquement
- design cohérent avec les autres widgets
```

# 📊 Signification des colonnes

* **Backlog** → idées pas encore structurées
* **Ready** → tâches bien définies, prêtes à coder
* **In Progress** → agent ou toi en train de coder
* **Review** → test + validation
* **Done** → terminé

# 💡 Philosophie du projet

* petites issues = meilleur résultat avec IA
* EPIC = organisation mentale
* ISSUE = travail réel
* Acceptance Criteria = définition du "done"
* GitHub Project = vision globale
